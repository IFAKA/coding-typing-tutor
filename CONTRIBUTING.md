# Contributing to coding-type

First off, thank you for considering contributing to `coding-type`! It's people like you that make the developer community such a great place.

## How Can I Contribute?

### Reporting Bugs

- **Check the existing issues** to see if the bug has already been reported.
- If not, **open a new issue**. Clearly describe the problem, including steps to reproduce it and your environment (OS, terminal, Go version).

### Adding Snippets

One of the best ways to contribute is by adding more algorithm snippets!
- Snippets are located in `internal/snippets/data/`.
- They are JSON files named by language (e.g., `python.json`).
- Each snippet should be a common algorithm pattern or a helpful coding interview template.

### Suggesting Enhancements

- Open an issue with the "enhancement" tag.
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style.
5. Issue that pull request!

## Style Guide

- We use standard Go formatting (`go fmt`).
- Keep snippets clean and well-commented where appropriate.
